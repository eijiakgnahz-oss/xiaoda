# 桌宠产品 · 小搭 🕹️

一只**私人助理桌宠**：会按作息睡觉/工作/吃零食/喝水，早中晚问好，提醒你吃饭休息；
点它弹出本机 CPU / 内存 / 存储 / 电池 / 网络 / Claude Code 信息面板；点「聊天」并配置自己的
OpenAI 兼容模型后可以：联网搜资讯/查名词/搜电影（OpenRouter 可自动路由到联网模型）、打开应用和网页、新建/移动文件（弹窗确认）、
记住你的名字喜好（长期记忆）、对话式设定时提醒（"每天3点提醒我喝水"）、
按你的职业每天上午 10 点强提醒一次行业情报（内容等你要看时才拉取）。

## 启动流程（角色系统）

1. **首次启动** → 弹出「选择你的小搭」角色页：3 个默认角色（小蓝/小粉/小绿）+ 透明加号格；
2. 点 **＋** 可上传任意图片，**本地像素化**（居中裁方 → 32×32 降采样 → 颜色量化，零联网、秒出）
   生成专属小人，确认后自动进入角色库并选中；自定义角色左上角 ✕ 可删除（默认角色不可删）；
3. 点 **确定** → 小人上桌面。此后启动会**记住上次选择**直接上桌面；
4. 想换角色：右键小人 → **更换角色**，重新打开选择页。

自定义角色的动作动画原理：单张像素图 + 道具叠加（工作=笔记本、零食/喝水=道具像素）+
上下位移（待机呼吸/挥手）+ 整体压暗（睡觉），见 `sprites.js` 的 `buildCustomFrames()`。

## 一、怎么运行

```bash
cd xiaoda
npm install          # 首次需要；国内建议加镜像：
                     # ELECTRON_MIRROR=https://npmmirror.com/mirrors/electron/ npm install --registry=https://registry.npmmirror.com
npm start
```

- 小人出现在屏幕右下角，**左键按住可拖动**，**单击**弹出信息面板，**右键**弹出退出菜单。
- 想让它能聊天：点小人 → 聊天 → ⚙，选择服务模板或填写任意 OpenAI 兼容接口，
  再填入你自己的 API Key 和模型名。模板只填 API 地址，模型名由用户自己决定。
  Key 存在你本机的用户配置目录里，不会进代码。

## 二、模型配置说明

小搭不会内置作者的模型账号或 API Key。每个用户都需要使用自己的模型服务。

### 不配置模型也能使用

- 桌宠动画：睡觉、工作、散步、喝水、吃零食、挥手；
- 主动陪伴：早中晚问好、饭点提醒、久坐休息提醒；
- 角色系统：选择内置角色、上传图片生成像素小人、切换/删除自定义角色；
- 桌面交互：拖拽、缩放、右键菜单；
- 系统信息面板：CPU、内存、存储、电池、网络、Claude Code 状态。

### 需要配置模型后才能使用

- AI 聊天、问答、查名词；
- 联网搜索、查资讯、查电影、今日情报；
- 打开应用、打开网页；
- 查看文件夹、新建文件、移动文件；
- 记住用户名字/习惯/喜好；
- 记录职业/兴趣；
- 通过对话设置、查看、删除提醒。

### 用户需要填写的三项

1. **API 地址**：模型服务的 OpenAI 兼容接口地址；
2. **模型名**：用户自己选择的模型名称；
3. **API Key**：用户在对应平台申请的密钥，只保存在本机配置目录。

## 三、项目结构 = 制作流程

| 文件 | 承担的模块 | 用到的技术/原理 |
|---|---|---|
| `main.js` | 主进程：透明置顶窗口、系统信息读取、用户自选模型对接、本机控制执行 | Electron BrowserWindow、`os`/`df`/`pmset` 本地命令、Function Calling |
| `preload.js` | 安全桥梁：界面只能用白名单接口 | Electron contextBridge（进程隔离） |
| `renderer/sprites.js` | 像素素材：32×32 街机风程序化绘制（自动阴影+描边），6 个动作 | 光源左上自动打阴影、轮廓自动包黑边，换素材只改这一个文件 |
| `renderer/app.js` | 动画引擎、作息状态机、问候/提醒/情报、面板、聊天 | Canvas 逐像素绘制、定时器状态机（无 AI） |
| `renderer/index.html` + `style.css` | 界面与像素风样式 | 透明窗口 + `image-rendering: pixelated` |

## 四、核心原理速览

### 1. 哪些地方有 AI，哪些没有
- **没有 AI**：睡觉/工作/吃零食/喝水的切换、早中晚问好、吃饭休息提醒——全部是
  「读系统时钟 + 定时器 + 状态机」，见 `renderer/app.js` 的 `tick()`。
- **有 AI**：只有聊天。用的是 **Function Calling（工具调用）**：
  1. 把小搭能做的事写成工具清单（`main.js` 里的 `TOOLS`）发给用户配置的 OpenAI 兼容模型；
  2. 模型不直接碰电脑，只返回结构化指令，如 `{"tool":"open_app","args":{"name":"备忘录"}}`；
  3. 本地代码执行（`executeTool`），结果回传模型，模型再用小搭口吻回复。

### 2. 联网能力与边界
- **联网搜索**：如果用户配置的是 OpenRouter，检测到搜索/资讯类意图时（`WEB_RE`），本次请求自动切换到
  `:online` 模型变体（自带网页搜索），平时用普通模型省成本；
- **电脑操作依然白名单**：模型只能调用注册表里的工具（开应用/开网页/文件操作/记忆/提醒），
  写文件类操作先弹系统确认框——安全靠代码强制，不靠提示词；
- 合规约束写入系统提示词：拒绝违法违规内容请求。

### 3. 文件操作安全
- 所有路径限制在**用户主目录内**（`insideHome`），系统目录一律拒绝；
- 新建/移动前弹**系统确认框**，你点「允许」才执行；
- 不覆盖已存在的文件。

### 4. 桌宠窗口的实现技巧
- 透明、无边框、置顶窗口 + `setIgnoreMouseEvents(true, {forward:true})` 让整个窗口对鼠标"隐形"；
- 渲染进程监听 `mousemove`，指到小人/面板（`.hit` 元素）上时才恢复鼠标接管——
  这是桌宠类应用的标准做法，否则透明区域会挡住你点击桌面。

## 五、已知取舍与后续路线
- 像素小人目前是**代码生成的占位素材**，正式形象只需替换 `sprites.js` 的网格；
- 内存显示用的是 `total - free`，macOS 上会偏高（系统缓存也算在内），够用但不精确；
- 打包成可分发的 `.app`：后续加 `electron-builder` 即可（`npm i -D electron-builder`）。
